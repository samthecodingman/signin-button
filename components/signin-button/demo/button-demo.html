Buttons can be generated from an inbuilt list of providers

  • Hovering over a signin-button will show the button's label.
  • An omitted or undefined provider will be shown as unknown.
  • These are the authentication methods used by Firebase.
  • More providers are available using signin-button-more-providers.

An expanded list of providers is also available

This is achieved using the signin-button-more-providers element, which wraps a signin-button and has more provider definitions.

Buttons also have a dark mode.

Buttons also have a short mode where only the button's name is shown.

This will also override the hover text.

Buttons also have a tile mode where only the button's icon is shown.

If this is used in combination with the short mode, the hover text will be changed. (Hover over the buttons below)

Buttons can be styled and text can be overridden using attributes

Note: All values will use their default implementation if the attribute value is interpreted as falsy.

name

The case-accurate name of this provider.

Notes:
  • shown on its own when short is true
  • concatenated with prefix when short is true
Example:
"Google", "SoundCloud"
Default:
calculated from provider
prefix

The text shown before name in the label.

Notes:
  • a trailing space is important for spacing if needed
  • hidden when attribute short is true
Example:
"Sign in: ", "Sign in to "
Default:
"Sign in with "
label

Overrides the text shown in the label and the button's hover text.

Notes:
  • hidden when attribute short is true
  • hidden when attribute tile is true
Example:
"Sign in", "Signed in"
Default:
concatenates prefix and name
logo-color

Overrides the primary color used by this signin-button

Notes:
  • value must be a hexadecimal color (either #xxx or #xxxxxx)
Example:
#f44336, #4caf50, #2196f3
Default:
calculated from provider
icon

Overrides the icon shown on the button.

Notes:
  • value must be a hexadecimal color (either #xxx or #xxxxxx)
Example:
"signin:twitter", "maps:local-activity" "maps:local-mall"
Default:
calculated from provider

To add your own provider, you can do so either as an object or using attributes.

new-provider

Used to create a new provider but is passed in as an object with the appropriate attribute values.

Type:
Object
Structure:
provider
ID used by login events
name
name of this provider
icon
icon identifier for this button
prefix
Text shown before name in the label.
label
Text to use as the label
Example:
#f44336, #4caf50, #2196f3
Default:
calculated from provider

Buttons can have their styles overridden using mixins if desired.

Note: Avoid changing the internal visual style (apart from colors) as the buttons already implement Google's sign-in branding guidelines. This mixin API is only provided to align with other Polymer Custom Elements.
